PPT Points With Explanation
Project Title & Vision
AI Resume & Job Matching System is a dual-mode platform for students and companies.
It helps students improve resume quality (ATS readiness) and helps recruiters rank candidates against a job description.

Problem Statement
Resume screening is often manual, slow, and inconsistent.
Students don’t know why resumes get rejected, and recruiters spend too much time shortlisting.

Core Objective
Build one system that:

analyzes a single resume for ATS quality, and
compares multiple resumes against a job role to find best matches.
Two User Flows (Key Feature Split)

Student Flow (Dashboard): upload one resume → get ATS score + detailed improvement tips.
Company Flow (For Company): upload JD + multiple resumes → get ranked list + downloadable report PDF.
Student ATS Engine (What Makes It Strong)
In Dashboard/atsScoreModel.js, scoring is not just random keywords; it checks structured factors:

contact details (email/phone/links),
standard sections (experience/education/skills/projects),
formatting signals (bullets, dates),
content depth and word diversity,
action verbs and quantified impact.
This gives a more explainable score.
AI Enrichment Layer
If OPENAI_API_KEY is available, ATS feedback is enriched with GPT-based personalized suggestions.
So the system can work in both modes: rule-based only and rule + AI guidance.

Company Matching Intelligence
In For Company/server.js + sbert_api.py, you use semantic matching:

resume text embedding,
job description embedding,
cosine similarity score (SBERT).
This is stronger than plain keyword matching because it captures meaning similarity.
Automated Recruiter Output
Company mode auto-generates:

ranked candidate table (name/email/phone/score),
resume download links,
consolidated PDF report for quick hiring decisions.
Tech Stack Used

Frontend: HTML, CSS, JavaScript
Backend: Node.js + Express
File/Text Processing: multer, pdf-parse, mammoth
AI/NLP: SentenceTransformers (Flask API), optional OpenAI enrichment
Database: MySQL (mysql2)
Security: bcryptjs, jsonwebtoken (company backend dependencies)
User Experience Highlights
Clean dashboards, drag-and-drop upload, progress feedback, and separate interfaces for student/company roles.
This makes the project easy to demo in a live presentation.

Business/Real-World Impact

Reduces recruiter screening effort.
Increases students’ interview readiness through actionable feedback.
Creates a bridge between employability improvement and recruitment efficiency.
Current Limitations (Good to Mention Honestly)

Mostly local setup (localhost endpoints).
Mixed authentication approach (part localStorage-based, part backend auth).
Hardcoded DB credentials in config (needs environment variables).
Mentioning these shows maturity in analysis.
Future Scope

Cloud deployment + role-based production auth.
Better OCR support for scanned PDFs.
JD-specific feedback for students.
Interview recommendation engine and analytics dashboard for recruiters.
